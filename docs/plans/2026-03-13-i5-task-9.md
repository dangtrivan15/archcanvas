## Description

### Keyboard Shortcuts (Contract C10)

New hook: `useAppKeyboard()` registered at `App.tsx` level (above canvas).

| Shortcut | Action |
|---|---|
| `Cmd+S` / `Ctrl+S` | `fileStore.save()` |
| `Cmd+O` / `Ctrl+O` | `fileStore.open()` |
| `Cmd+Shift+S` / `Ctrl+Shift+S` | `fileStore.saveAs()` |

**Important:** Ignore events when active element is `<input>`, `<textarea>`, or `contentEditable="true"` (C10.5). Use `useEffect` with `keydown` listener, check `e.metaKey || e.ctrlKey`, `e.preventDefault()` to suppress browser defaults (especially Cmd+S save dialog).

Pattern should follow existing `useCanvasKeyboard` hook style but at a higher level.

### Dirty Indicator (Contract C8)

**StatusBar.tsx:**
- Read `fileStore.isDirty()` reactively
- When dirty: show "Modified" indicator (amber/yellow badge)
- When clean: no indicator (C8.3)
- Updates reactively (C8.4)

**App.tsx — Title:**
- `useEffect` watching `isDirty()` and project name
- Dirty: `"● {projectName} — ArchCanvas"`
- Clean: `"{projectName} — ArchCanvas"`
- No project: `"ArchCanvas"`

### Save-on-Close (Contract C9)

**Web (beforeunload):**
- In App.tsx: `window.addEventListener('beforeunload', handler)`
- If `isDirty()`, set `e.returnValue = true` to trigger native browser dialog
- Browser shows generic "Changes may not be saved" dialog (custom text not supported)
- If not dirty, handler is no-op (close proceeds normally)

**Tauri:**
- Listen for Tauri close-requested event
- If dirty: show custom dialog with three options (Save & Close, Discard, Cancel)
- For now, Tauri support can be a stub/placeholder since Tauri is scaffold-only
