## Description

This task removes the fragile DOM event pattern for container navigation and updates
the core canvas hooks to use the unified navigation store directly.

### ContainerNode: Replace DOM Event

**Before:**
```typescript
const handleDiveIn = useCallback(() => {
  document.dispatchEvent(new CustomEvent('archcanvas:container-dive-in', {
    detail: { nodeId: nodeData.archNodeId, refSource: nodeData.refSource },
    bubbles: true,
  }));
}, [nodeData.archNodeId, nodeData.refSource]);
```

**After:**
```typescript
const handleDiveIn = useCallback(() => {
  // Import the dive-in action from navigation store or pass as prop
  // The exact mechanism depends on whether ContainerNode should import stores directly
  // Option A: Import useNavigationStore and call diveIntoNode/diveIntoFile
  // Option B: Call a callback prop passed from Canvas
  // Recommendation: Option A is simpler and matches how other components work
  useNavigationStore.getState().diveIntoNode(nodeData.archNodeId);
}, [nodeData.archNodeId]);
```

**Decision needed:** ContainerNode currently uses DOM events specifically to avoid
importing stores. However, many React Flow custom nodes already import stores
(NodeContextMenu, etc.). Using `.getState()` is fine for event handlers — it's not
a render-time subscription.

For `refSource` (cross-file dive), the logic in `useContainerDiveIn.performDiveInSwitch()`
that loads the file and calls `pushFile()` needs to stay. The ContainerNode should
call a navigation action that handles both cases (node dive vs. file dive).

### useCanvasNavigation: Remove Event Listener

Remove the `document.addEventListener('archcanvas:container-dive-in', ...)` block
(lines 64-73). The dive-in is now triggered directly from ContainerNode → store.

Simplify the hook to only handle:
1. Escape key zoom-out / nested file exit
2. Ctrl+Shift+Home jump to root
3. Enter key on selected container → dive-in
4. Pinch gesture → zoom out
5. Auto-layout on navigation depth change

Update all store references from old API to unified API:
- `useNavigationStore(s => s.path)` → keep (backwards-compatible alias)
- `useNestedCanvasStore.getState().popFile()` → `useNavigationStore.getState().goUp()`
- `useNestedCanvasStore.getState().popToRoot()` → `useNavigationStore.getState().goToRoot()`

### useContainerDiveIn: Update Store Calls

Replace:
- `nestedStore.pushFile(filePath, graph, containerNodeId, containerColor)` → `navigationStore.diveIntoFile(filePath, graph, containerNodeId, containerColor)`
- `nestedStore.popFile()` → `navigationStore.goUp()`

The animation phases stay the same — only the store calls change.

### NavigationBreadcrumb: Use Unified Breadcrumb

**Before:** Reads `path: string[]` and resolves node names by traversing graph.

**After:** Reads `breadcrumb: BreadcrumbEntry[]` which already has `displayName`.
Also shows file entries in the breadcrumb (currently invisible since only
navigationStore's `path` is used).

Update rendering to handle both `type: 'node'` and `type: 'file'` entries.
File entries might show a file icon or different styling.

### NestingFrame: Read Unified Store

Replace `useNestedCanvasStore(s => s.fileStack)` with unified store reads.
The `fileDepth` and `nodeDepth` are now directly available as computed properties.
