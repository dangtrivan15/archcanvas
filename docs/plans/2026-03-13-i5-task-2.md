## Description

The CLI runs in Node.js and needs a `FileSystem` implementation backed by `node:fs/promises`. The `createFileSystem` factory must detect the runtime environment and return the appropriate implementation.

### NodeFileSystem (Contract C2)

Implements all 5 `FileSystem` methods:
- `readFile(path)` ‚Äî `fs.readFile(resolve(root, path), 'utf-8')`
- `writeFile(path, content)` ‚Äî `fs.mkdir(dirname, { recursive: true })` then `fs.writeFile`
- `listFiles(dir)` ‚Äî `fs.readdir(resolve(root, dir))` returning filenames only
- `exists(path)` ‚Äî `fs.access().then(() => true, () => false)` (never throws)
- `mkdir(path)` ‚Äî `fs.mkdir(resolve(root, path), { recursive: true })`

All paths resolved relative to `root` via `node:path.resolve()`.

### Factory Update (Contract C1)

Detection order (critical ‚Äî Tauri runs in a webview with `window`):
1. `FileSystemDirectoryHandle` ‚Üí `WebFileSystem` (any environment)
2. `typeof window === 'undefined'` ‚Üí `NodeFileSystem` (dynamic import to avoid bundling `node:fs`)
3. `window.__TAURI_INTERNALS__` present ‚Üí `TauriFileSystem`
4. None of the above ‚Üí throw `Error('Unsupported environment')`

Key invariant: `NodeFileSystem` is loaded via `await import('./nodeFileSystem')` ‚Äî never statically imported ‚Äî so it doesn't enter the browser bundle (C1.6).

### Testing Strategy

**nodeFileSystem.test.ts:** Real temp directory (`fs.mkdtemp`) ‚Äî no mocks. Tests all 5 methods including edge cases (missing file throws, parent directory auto-creation, exists returns false not throw).

**createFileSystem.test.ts:** Mock `window` global presence/absence and `__TAURI_INTERNALS__`. Use `vi.importActual` to verify dynamic import path. Mock `FileSystemDirectoryHandle` with a class stub.

**Acceptance criteria:**
- C1.1‚ÄìC1.6: Factory returns correct implementation for each environment
- C2.1‚ÄìC2.6: All NodeFileSystem methods work correctly with real filesystem
