## Description

Three read-only commands + one mutation command (import).

### list (Contract C5f)

Flags: `--scope` (default root), `--type` (`nodes|edges|entities|all`, default `all`)

1. `loadContext(--project)`
2. `fileStore.getCanvas(resolveCanvasId(scope))` ‚Üí get canvas data
3. If scope doesn't exist ‚Üí `CANVAS_NOT_FOUND`
4. Filter by `--type` if specified
5. Human output: formatted table with columns per type. JSON: `{ nodes: [...], edges: [...], entities: [...] }`

### describe (Contract C5g)

Flags: `--id` (optional), `--scope` (default root)

Two modes:
- **Node detail** (`--id` provided): Display type, displayName, args, connected edges, notes, codeRefs. Ports resolved from NodeDef registry (not stored on node).
- **Architecture summary** (no `--id`): Project name, node/edge/entity counts per scope. For ref nodes, include child summary (counts inside subsystem canvas).

### search (Contract C5h)

Positional: `<query>` (required). Flags: `--type` filter.

1. Iterate ALL entries in `fileStore.project.canvases` (only loaded/reachable canvases)
2. For each canvas: match query against node IDs, displayNames, types, entity names, edge labels
3. Case-insensitive substring matching (not fuzzy)
4. Results include scope (canvasId) where match was found
5. JSON: `{ results: [{ type, scope, item }] }`

### import (Contract C5i)

Flags: `--file` (required path to YAML), `--scope` (default root)

**This is the only mutation command in this task.**

1. Read YAML file using `node:fs/promises` (or the CLI's FileSystem)
2. Parse as partial canvas: arrays of nodes, edges, entities
3. Iterate: call `graphStore.addNode/addEdge/addEntity` for each item
4. Collect errors per item ‚Äî do NOT stop on first error (C5i.3)
5. Save once at end: `fileStore.saveAll(ctx.fs)` (C5i.4)
6. Output: summary of added/failed counts

**YAML parsing:** Uses the `yaml` package (already a project dependency).

### Testing Strategy

Common setup: Create InMemoryFileSystem with a populated project (nodes, edges, entities across multiple scopes). Call handlers directly.

- **list:** Verify filtering by type, table output format, JSON shape, CANVAS_NOT_FOUND error
- **describe:** Node detail with ports from registry, architecture summary with scope counts, ref node child summary
- **search:** Cross-scope search, case insensitivity, type filtering, scope in results
- **import:** Successful import, partial errors collected, single save at end, JSON counts
