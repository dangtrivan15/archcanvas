## Description

This is the core task of I6b — it changes how the app handles empty/missing projects.

### Part 1: `needs_onboarding` Detection

Modify `openProject(fs)` to detect empty-project state **before** calling `loadProject()`:

1. Check `fs.exists('.archcanvas')` — if false → `set({ fs, status: 'needs_onboarding' })`
2. Check `fs.exists('.archcanvas/main.yaml')` — if false → `set({ fs, status: 'needs_onboarding' })`
3. Read `main.yaml` content — if `content.trim() === ''` → `set({ fs, status: 'needs_onboarding' })`
4. Otherwise, proceed with `loadProject()` as before

Critical: `fs` and `status` must be set in the **same `set()` call** to avoid routing race conditions.

Add `'needs_onboarding'` to the `FileStoreState.status` union type.

### Part 2: Simplify `newProject()`

Currently `newProject()` scaffolds `.archcanvas/main.yaml` and calls `openProject()`. Change to:
1. Pick directory (via FilePicker)
2. Call `openProject(fs)` — which detects no `.archcanvas/` → sets `needs_onboarding`

Remove the scaffolding code from `newProject()`.

### Part 3: `completeOnboarding` Action

New store action:

```typescript
interface SurveyData {
  description: string;
  techStack: string[];
  explorationDepth: 'full' | 'top-level' | 'custom';
  customDepth?: number;
  focusDirs: string;
}

completeOnboarding: async (type: 'blank' | 'ai', survey?: SurveyData) => {
  const { fs } = get();
  if (!fs) return;

  const name = fs.getName();

  // 1. Scaffold .archcanvas/
  if (!(await fs.exists('.archcanvas'))) {
    await fs.mkdir('.archcanvas');
  }

  // 2. Write main.yaml
  const description = type === 'ai' && survey ? survey.description : '';
  const mainYaml = serializeCanvas({
    project: { name, description, version: '1.0.0' },
    nodes: [],
    edges: [],
    entities: [],
  });
  await fs.writeFile('.archcanvas/main.yaml', mainYaml);

  // 3. Load the project
  await get().openProject(fs);

  // 4. Update recents (only if loaded successfully)
  if (get().status === 'loaded') {
    set({ recentProjects: addToRecent(get().recentProjects, name, name) });
  }

  // 5. AI path: open chat + send init prompt
  if (type === 'ai' && survey && get().status === 'loaded') {
    useUiStore.getState().toggleChat();
    // Wait for next tick so assembleContext() reads valid project state
    setTimeout(() => {
      const prompt = assembleInitPrompt(name, survey);
      useChatStore.getState().sendMessage(prompt);
    }, 0);
  }
}
```

Note: `assembleInitPrompt` is created in Task 3. For this task, import it and use it — if Task 3 isn't done yet, create a placeholder that returns a simple string. The implementer should check if the function exists and create a stub if needed.

### Part 4: Modify `open()` behavior

No code changes needed to `open()` itself — it calls `openProject()`, which now handles the detection. But verify that `open()` stores `fs` correctly: currently it only sets `fs` when `status === 'loaded'`. With the new detection in `openProject()`, `fs` is set inside `openProject()` when `needs_onboarding` is detected, so `open()` should NOT overwrite it.

Check the current `open()` code:
```typescript
if (get().status === 'loaded') {
  set({ fs, ... });
}
```
This guard means `fs` won't be set by `open()` on `needs_onboarding`. But `openProject()` already sets `fs` in `set({ fs, status: 'needs_onboarding' })`. So `open()` just needs to not interfere — verify this works correctly.
