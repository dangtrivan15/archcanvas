## Description

This task creates the EntityPanel React component and integrates it into the right panel area as a third tab alongside NodeDetailPanel/EdgeDetailPanel and ChatPanel.

**EntityPanel layout:**
- Header: "Entities" with current scope name (from `navigationStore`)
- Search/filter input: filters entity list by name (client-side, immediate)
- Entity list: each row shows entity name, truncated description, usage badge (edge count in current scope)
- Expandable detail: click to expand → full description, codeRefs, cross-scope usages via `findEntityUsages()`
- Cross-scope usage items show canvas name + edge count; clicking navigates to that canvas

**RightPanel integration:**

Currently `RightPanel.tsx` switches between `'details'` and `'chat'` based on `uiStore.rightPanelMode`. Add `'entities'` as a third mode. Add a tab/button for switching to entities view. The panel should show:
- `'details'` → NodeDetailPanel or EdgeDetailPanel (existing)
- `'chat'` → ChatPanel (existing)
- `'entities'` → EntityPanel (new)

**uiStore change:**

```typescript
// Before
rightPanelMode: 'details' | 'chat'

// After
rightPanelMode: 'details' | 'chat' | 'entities'
```

**Data flow:**
1. EntityPanel reads current scope from `navigationStore`
2. Calls `getEntitiesForCanvas(project, currentCanvasId)` for the scoped list
3. On expand, calls `findEntityUsages(project, entityName)` for cross-scope data
4. Project data comes from `fileStore.project`

**Acceptance criteria:**
- EntityPanel renders entities from current scope
- Search filters entities by name
- Expanding an entity shows full detail + cross-scope usages
- RightPanel has three modes: details, chat, entities
- Tab switching between modes works correctly
- Empty state shown when no entities in current scope
