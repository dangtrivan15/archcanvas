## Description

This task turns the Tauri scaffold into a real desktop app with the AI bridge running as a sidecar process.

**Step 1: Risk gate — validate `bun build --compile`**

Before any Tauri work, verify that `bun build --compile dist/bridge-server.js` produces a working binary:
1. Run `bun build --compile dist/bridge-server.js --outfile /tmp/archcanvas-bridge`
2. Run `/tmp/archcanvas-bridge --port 17248 --cwd /tmp/test`
3. Test WebSocket connection and HTTP routes

If this fails (native dependency issues, SDK incompatibility):
- **Fallback**: ship `dist/bridge-server.js` as a script sidecar, add Node.js as a prerequisite in docs
- Document what broke for future resolution

**Step 2: Tauri sidecar configuration**

In `tauri.conf.json`:
```json
{
  "bundle": {
    "externalBin": ["binaries/archcanvas-bridge"]
  }
}
```

Tauri expects platform-specific binary names: `archcanvas-bridge-aarch64-apple-darwin` for Apple Silicon, `archcanvas-bridge-x86_64-apple-darwin` for Intel. The `build:sidecar` script must produce the correct name.

**Step 3: Rust sidecar management**

In `main.rs`:
1. On app launch, spawn the sidecar using Tauri's `Command::new_sidecar("archcanvas-bridge")`
2. Pass `--cwd` argument (project directory — resolved from app state or file dialog)
3. Read stdout line-by-line; parse first line for `BRIDGE_PORT=<number>`
4. Store port in app state
5. Expose `get_bridge_port` as a Tauri command: `#[tauri::command] fn get_bridge_port(state) -> u16`
6. On app close, kill the sidecar process

**Step 4: Frontend port discovery**

In `webSocketProvider.ts`, add a port resolution step before connecting:
```typescript
async function resolveBridgePort(): Promise<number> {
  if (window.__TAURI__) {
    return await invoke('get_bridge_port');
  }
  return parseInt(import.meta.env.VITE_BRIDGE_PORT ?? '17248');
}
```

**Step 5: Tauri packaging**

- Install `@tauri-apps/plugin-fs` npm package (remove custom `.d.ts` workaround)
- Configure app signing (Apple Developer ID) — this requires your signing certificate
- Run `npm run tauri build` to produce `.dmg`

**Build pipeline (`package.json`):**
```json
{
  "build:sidecar": "bun build --compile dist/bridge-server.js --outfile src-tauri/binaries/archcanvas-bridge-aarch64-apple-darwin"
}
```

**Acceptance criteria:**
- `bun build --compile` produces a working binary (or fallback is documented)
- Tauri app launches and spawns the sidecar
- Sidecar port is captured and available to the webview
- WebSocket connection works in the Tauri app
- `npm run tauri build` produces a `.dmg`
- AI chat works in the packaged Tauri app
