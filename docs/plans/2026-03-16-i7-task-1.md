## Description

This task establishes the data model foundation for the cross-scope ref redesign. Three changes:

1. **Schema**: `SubsystemCanvas` refinement currently requires `id && type`. Change to require only `type`. Subsystem files no longer need an `id` field — their identity comes from the parent ref-node's `id`.

2. **fileResolver ref handling**: Currently `fileResolver.ts:81` appends `.yaml` to the ref value when constructing file paths. Remove this — `ref` now includes the `.yaml` suffix explicitly (e.g., `ref: svc-order.yaml`). Also change the canvases map key from `ref` (the filename) to `node.id` (the ref-node identity). This is critical: `canvases.set(node.id, loadedCanvas)` not `canvases.set(ref, loadedCanvas)`. All downstream lookups already use the ref-node id, so this change is transparent to consumers.

3. **Cross-scope validation**: Replace `validateRootRefs()` (which only validates `@root/` against root canvas) with `validateCrossScopeRefs()`. The new function iterates all loaded canvases, finds edges with `@`-prefixed endpoints, parses `@<ref-node-id>/<node-id>`, resolves `ref-node-id` to a canvas via the canvases map, and checks that `node-id` exists in that canvas's nodes. If the referenced canvas failed to load (error during `resolveRefs`), flag as `CROSS_SCOPE_REF_UNRESOLVABLE` (degraded, not crash). Note: `@<ref-node-id>` resolution for engine-time checks (does the ref-node exist in this canvas?) is handled in Task 2, not here.

**Edge case**: YAML quoting — `@` is a reserved indicator in YAML. The spec notes that cross-scope ref values must be quoted: `node: "@order-service/processor"`. Existing `@root/` values are already quoted in practice, so this is not a new concern, but tests should verify quoted strings parse correctly.

**Acceptance criteria:**
- `SubsystemCanvas` validates successfully without `id` field
- `ref: svc-order.yaml` resolves to `.archcanvas/svc-order.yaml` (no double `.yaml`)
- Canvases map is keyed by ref-node `id`, not filename
- Cross-scope edges like `@order-service/processor` are validated against the referenced canvas
- Invalid cross-scope refs produce `CROSS_SCOPE_NODE_NOT_FOUND` errors
- Unresolvable refs (canvas load failure) produce `CROSS_SCOPE_REF_UNRESOLVABLE` errors
- All existing fileResolver tests pass with updated fixtures
