## Description

These tests run the **built** `dist/cli.js` binary in a temp directory. They verify the full end-to-end flow: argument parsing, context loading, store operations, file I/O, and output formatting.

**IMPORTANT:** Use `node:child_process` `execFile` (NOT `exec`) to avoid shell injection. Wrap in a Promise helper for clean async usage.

### Test Flow

Each test:
1. Create a temp directory via `node:fs/promises` `mkdtemp`
2. Spawn `node dist/cli.js <command> <flags>` safely
3. Assert stdout/stderr content and exit code
4. Verify filesystem state (files created/modified)
5. Cleanup temp directory in afterEach

### Helper Pattern

```typescript
// Use execFile for safety (no shell injection)
import { execFile as execFileCb } from 'node:child_process';
import { promisify } from 'node:util';
const execFileAsync = promisify(execFileCb);

async function runCLI(args: string[], cwd: string) {
  try {
    const { stdout, stderr } = await execFileAsync('node', ['dist/cli.js', ...args], { cwd });
    return { stdout, stderr, exitCode: 0 };
  } catch (err: any) {
    return { stdout: err.stdout ?? '', stderr: err.stderr ?? '', exitCode: err.code ?? 1 };
  }
}
```
