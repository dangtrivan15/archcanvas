## Description

### init Command (Contract C5a)

`archcanvas init` creates a new project directory structure. Unlike other commands, init does NOT use `loadContext()` — there's no existing project to load. It creates files directly via `FileSystem`.

**Flags:**
- `--name <string>` — project name (defaults to directory name)
- `--path <string>` — target directory (defaults to cwd)

**Flow:**
1. `createFileSystem(targetPath)` → get FileSystem
2. Check if `.archcanvas/main.yaml` already exists → error `PROJECT_EXISTS` if so
3. `fs.mkdir('.archcanvas')`
4. `fs.writeFile('.archcanvas/main.yaml', template)` — template with project metadata + empty arrays

**Template:**
```yaml
project:
  name: <name>
  description: ""
  version: "1.0.0"

nodes: []
edges: []
entities: []
```

**Output:**
- Human: `"Initialized project '<name>' at <path>"`
- JSON: `{ ok: true, project: { name, path } }`

### Vite CLI Build Config

`vite.config.cli.ts` — separate Vite config for building the CLI as a Node.js bundle:
- Entry: `src/cli/index.ts`
- Target: `node` (via `ssr` or `build.target`)
- Output: `dist/cli.js` (single file)
- Format: ESM (with `#!/usr/bin/env node` banner)
- Externals: Node.js builtins (`node:fs`, `node:path`, etc.)
- Bundles: `commander`, `yaml`, `zod`, `immer`, `zustand`, all `src/` imports

Add `build:cli` script to `package.json`: `"build:cli": "vite build --config vite.config.cli.ts"`

### Testing Strategy

**init.test.ts:** Use temp directories (real filesystem, not InMemory — init creates actual files). Test: directory creation, main.yaml template correctness, PROJECT_EXISTS error, default name from directory, custom name/path, JSON output shape.
