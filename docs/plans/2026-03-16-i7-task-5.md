## Description

This task adds protocol compatibility checking to edge creation. When an edge specifies ports on both endpoints, the engine resolves the port definitions from the NodeDef registry and checks whether their protocol arrays have any overlap. If they don't, the edge is rejected with a hard error.

**`arePortsCompatible()` function:**

```typescript
interface ProtocolCheckResult {
  compatible: boolean;
  fromPortName?: string;
  fromProtocols?: string[];
  toPortName?: string;
  toProtocols?: string[];
}

function arePortsCompatible(fromPort: PortDef | undefined, toPort: PortDef | undefined): ProtocolCheckResult
```

Behavior:
- If either port is undefined → `{ compatible: true }` (skip check)
- If either port's protocol array is empty/undefined → `{ compatible: true }` (skip check)
- If any protocol appears in both arrays → `{ compatible: true }` with port details
- If no overlap → `{ compatible: false }` with both port names and protocol arrays

**Engine integration in `addEdge()`:**

After the existing endpoint existence check, if both endpoints specify ports:
1. Look up the source node's NodeDef from the registry
2. Find the `PortDef` matching the edge's `from.port` name
3. Same for destination: look up NodeDef, find `PortDef` for `to.port`
4. Call `arePortsCompatible(fromPortDef, toPortDef)`
5. If `compatible: false` → return `{ ok: false, error: { code: 'PROTOCOL_MISMATCH', message: ... } }`

**Edge cases:**
- Node has no NodeDef in registry (custom/unknown type) → skip check (no port info available)
- Port name specified but not found in NodeDef → existing `PORT_NOT_FOUND` warning handles this
- Cross-scope endpoints (`@ref-node-id/node`) → still resolve the inner node's type for port lookup

**Acceptance criteria:**
- `addEdge` with compatible ports succeeds (HTTP→HTTP)
- `addEdge` with incompatible ports fails (HTTP→SQL) with `PROTOCOL_MISMATCH`
- `addEdge` without ports specified succeeds (no check)
- `addEdge` with only one port specified succeeds (partial, no check)
- CLI `add-edge` shows meaningful error for protocol mismatch
- Error message includes port names and protocol arrays
