### Part 2: E2E Helpers

Add a helper for setting up an empty-project state for E2E tests:

```typescript
/**
 * Set up the app in needs_onboarding state (simulating opening an empty directory).
 * The wizard should appear instead of the canvas.
 */
export async function gotoEmptyProject(page: Page): Promise<void> {
  await page.goto('/');
  await page.evaluate(() => {
    const store = (window as any).__archcanvas_fileStore__;
    // Set fs with a stub + needs_onboarding status
    store.setState({
      fs: {
        getName: () => 'test-project',
        readFile: async () => '',
        writeFile: async () => {},
        exists: async () => false,
        mkdir: async () => {},
        listFiles: async () => [],
      },
      status: 'needs_onboarding',
      project: null,
      error: null,
    });
  });
  await page.waitForTimeout(100);
}
```

### Part 3: E2E Tests

`test/e2e/onboarding.spec.ts`:

1. **Wizard appears for empty project**: `gotoEmptyProject` → wizard is visible, canvas is not
2. **Blank canvas path**: wizard → click "Blank Canvas" → canvas loads, wizard disappears
3. **AI Analyze advances to survey**: wizard → click "AI Analyze" → Step 2 survey form visible
4. **Back from survey**: Step 2 → click "Back" → Step 1 visible again
5. **Existing project skips wizard**: `gotoApp` (normal setup) → canvas visible, no wizard

Note: Full AI init E2E (Step 2 → Start → chat opens with prompt) is not testable in E2E because `vite preview` has no AI bridge. The unit tests in Task 2 and Task 3 cover this path.
