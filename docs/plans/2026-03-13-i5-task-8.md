## Description

### fileStore Extensions (Contract C7)

Add to existing fileStore state:

```typescript
// New state
fs: FileSystem | null;              // active FileSystem instance
recentProjects: RecentProject[];    // last 5 opened paths

// New methods
open(): Promise<void>;              // picker → load project
save(): Promise<void>;              // delegates to saveAll(this.fs)
saveAs(): Promise<void>;            // picker → save to new location
isDirty(): boolean;                 // dirtyCanvases.size > 0
```

**`open()` flow:**
1. Call `filePicker.pickDirectory()` → returns `FileSystem | null`
2. If null (user cancelled), return
3. Call existing `openProject(fs)`, wait for `status === 'loaded'`
4. Set `this.fs = fs`
5. Update `recentProjects` (add/move to front, dedup by path, max 5)

**`save()` flow:**
1. If `this.fs` is null → fall through to `saveAs()` (C7.3)
2. Call existing `saveAll(this.fs)` (C7.2)

**`saveAs()` flow:**
1. Call `filePicker.pickDirectory()` → get new FileSystem
2. If null (cancelled), return
3. Call `saveAll(newFs)`
4. Replace `this.fs = newFs`
5. Update `recentProjects`

**`isDirty()` :** Simply `dirtyCanvases.size > 0` (C7.7).

### RecentProjects

```typescript
interface RecentProject {
  name: string;
  path: string;       // fs path or handle name (web: informational only)
  lastOpened: string;  // ISO timestamp
}
```

- Persisted to `localStorage` under key `archcanvas:recentProjects` (C7.5)
- Restored on store initialization
- Max 5 entries, most-recent first, deduped by path (C7.6)
- Web entries are informational only — handles not restorable from localStorage (C7.9)

### FilePicker Abstraction

```typescript
// src/platform/filePicker.ts
interface FilePicker {
  pickDirectory(): Promise<FileSystem | null>;  // null = user cancelled
}
```

- **Web:** Uses `showDirectoryPicker()` API → wraps result in `WebFileSystem`
- **Tauri:** Uses Tauri dialog API → wraps result in `TauriFileSystem`
- Returns `null` on cancel (user dismissed picker)
- Detect environment using same pattern as `createFileSystem`

### Testing Strategy
