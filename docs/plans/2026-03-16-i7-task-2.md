## Description

This task updates the graph engine, query, and validation layers to use the new `@<ref-node-id>/<node-id>` cross-scope syntax, replacing all `@root/` handling.

**Engine changes in `addEdge()`:**

Currently, the engine skips endpoint existence checks for `@root/` prefixed nodes (lines ~115-136). Replace with:
1. Parse `@`-prefixed endpoint: extract `refNodeId` and `nodeId` from `@refNodeId/nodeId`
2. Validate that `refNodeId` matches a ref-node `id` in the current canvas's nodes list. If not ŌåÆ `CROSS_SCOPE_REF_NOT_FOUND` error.
3. Skip the local node existence check (the fileResolver validates at load time that the node exists in the referenced canvas ŌĆö Task 1).
4. Self-loop check: `@x/a` ŌåÆ `@x/a` is a self-loop (same raw string comparison as before, just with new syntax).

**Engine changes in `removeNode()`:**

Currently removes edges where `from.node === nodeId` or `to.node === nodeId`. Add: also remove edges where any endpoint starts with `@<nodeId>/`. This handles the cascade when a ref-node is deleted ŌĆö all cross-scope edges reaching into that subsystem are removed.

**Query changes:**

`matchesNodeId()` currently handles `@root/<id>`. Replace: parse `@<anything>/<id>` and match the trailing `<id>` against the search target. This enables `findEdgesReferencingNode('db-postgres')` to match both `db-postgres` and `@order-service/db-postgres` (wait ŌĆö the second case shouldn't match; `db-postgres` in `@order-service/db-postgres` refers to a node named `db-postgres` inside order-service, not the root-level db-postgres). Actually, `matchesNodeId` should match `@root/db-postgres` ŌåÆ `db-postgres` in the current code. Under the new model, cross-scope edges live at the parent level, so `@order-service/processor` is an edge in the parent canvas referencing `processor` inside `order-service`. The query function should: (a) match exact string equality, (b) match when `@<anything>/<nodeId>` and the search target is `nodeId` AND the context canvas contains that ref-node. Re-examine the query use cases carefully and adjust.

**Acceptance criteria:**
- `addEdge` with `@order-service/processor` succeeds when `order-service` is a ref-node in the canvas
- `addEdge` with `@nonexistent/processor` fails with `CROSS_SCOPE_REF_NOT_FOUND`
- `removeNode('order-service')` cascades to remove edges with `@order-service/*` endpoints
- Self-loop `@x/a` ŌåÆ `@x/a` is rejected
- All existing engine/query/validation tests pass with updated fixtures
