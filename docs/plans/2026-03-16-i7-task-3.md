## Description

This task implements the render-time injection of inherited edges when a user dives into a subsystem canvas.

**Core mechanism in `useCanvasRenderer`:**

When the navigation scope is a child canvas (breadcrumb depth > 1):
1. Get the parent canvas from the canvases map using `navigationStore.parentCanvasId`
2. Scan parent canvas edges for any endpoint matching `@<current-ref-node-id>/*`
3. For each matched edge:
   - Strip the `@<ref-node-id>/` prefix from the matching endpoint → this becomes a local node reference
   - The OTHER endpoint (which is a node in the parent scope, not the child) becomes a **ghost node placeholder**
   - Add the edge to the ReactFlow edge list with `data: { inherited: true }`
4. Create ghost node entries for non-local endpoints:
   - Fixed size (e.g., 150x40)
   - Display: parent node's displayName + "(parent scope)" label
   - Style: dashed border, muted colors
   - Not interactive (no selection, no drag, no context menu)

**Ghost nodes in ELK layout:**

Ghost nodes participate in the ELK layout as regular nodes with fixed dimensions. This ensures inherited edges don't overlap with real nodes. They are positioned by ELK alongside real nodes.

**navigationStore changes:**

`diveIn()` currently does `const targetCanvasId = node.ref` at line 41. Change to use the `refNodeId` parameter directly (which is already the ref-node `id`). Also store `parentCanvasId` and `parentEdges` in the navigation state so `useCanvasRenderer` can access them without re-traversing the store graph.

**EdgeRenderer changes:**

When `data.inherited` is true, render with the existing ghost/dashed style pattern (`.ghost-marker` class). Add a hover tooltip showing "Connection from parent scope: [parent canvas name]". Inherited edges are not selectable or editable.

**Critical invariant:** Inherited edges and ghost nodes are NEVER included in save operations. They exist only in the ReactFlow render list, not in the graph store or file store.

**Acceptance criteria:**
- Diving into a subsystem shows inherited edges with ghost styling
- Ghost node placeholders appear for non-local endpoints with correct labels
- ELK layout positions ghost nodes alongside real nodes
- Inherited edges are not selectable or editable
- Going back to parent scope removes inherited edges from view
- Save operations do not include inherited edges
