## Description

This task wires ChatPanel into the existing app shell. It's small (~50 lines of changes across 4 files) but touches the UI integration seams.

### `src/store/uiStore.ts` Changes

Add `rightPanelMode` as **Zustand state** (not a module-level ref) because `RightPanel` needs to subscribe to it for reactive switching:

```typescript
interface UiState {
  // existing...
  rightPanelMode: 'details' | 'chat'
  setRightPanelMode: (mode: 'details' | 'chat') => void
  toggleChat: () => void  // toggles between 'details' and 'chat', opens panel if collapsed
}
```

`toggleChat()` implementation:
- If `rightPanelMode === 'chat'`, set to `'details'`
- If `rightPanelMode === 'details'`, set to `'chat'` AND call `openRightPanel()` (existing method) to expand the panel if collapsed

### `src/components/layout/RightPanel.tsx` Changes

Add a conditional at the top of the render:

```typescript
const rightPanelMode = useUiStore(s => s.rightPanelMode)
if (rightPanelMode === 'chat') {
  return <ChatPanel />
}
// ...existing selection-driven detail panel logic
```

Import `ChatPanel` from Task 4.

### `src/components/layout/LeftToolbar.tsx` Changes

Add a new tool config entry at the bottom of the tools array (below the divider, similar to how "Fit View" is positioned):

```typescript
{
  icon: MessageSquare,  // from lucide-react
  label: 'AI Chat',
  shortcut: '⌘⇧I',
  onClick: () => useUiStore.getState().toggleChat()
}
```

The button should visually indicate when chat mode is active (e.g., highlighted background like the active tool mode buttons).

### `src/components/hooks/useAppKeyboard.ts` Changes

Add `Cmd+Shift+I` handler:

```typescript
if (e.metaKey && e.shiftKey && e.key === 'I') {
  e.preventDefault()
  useUiStore.getState().toggleChat()
}
```

### Custom Event

Add an `archcanvas:toggle-chat` event listener (in Canvas.tsx or App.tsx) that calls `useUiStore.getState().toggleChat()`. This allows other components (e.g., command palette actions) to toggle chat without importing the store directly.

### Acceptance Criteria

- Clicking the AI Chat toolbar button toggles the right panel between details and chat mode
- `Cmd+Shift+I` toggles chat panel
- Chat button highlights when chat mode is active
- Opening chat auto-expands the right panel if it was collapsed
- Closing chat (via close button in ChatPanel or toolbar toggle) returns to details mode
- Node/edge selection in details mode works as before (no regression)
