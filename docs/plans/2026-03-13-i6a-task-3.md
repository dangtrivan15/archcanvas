## Description

This task builds the browser-side state layer: the WebSocket provider that talks to the bridge, and the Zustand store that manages conversation state.

### `src/core/ai/webSocketProvider.ts`

Implements the `ChatProvider` interface as a WebSocket client:

```typescript
class WebSocketClaudeCodeProvider implements ChatProvider {
  readonly id = 'claude-code'
  readonly displayName = 'Claude Code'

  get available(): boolean  // WebSocket.OPEN check

  connect(url: string): void
  sendMessage(content: string, context: ProjectContext): AsyncIterable<ChatEvent>
  loadHistory(messages: ChatMessage[]): void
  abort(): void
}
```

Key implementation details:
- **`connect(url)`**: Establishes WebSocket to `ws://localhost:5173/__archcanvas_ai`. Auto-reconnect on disconnect with exponential backoff (1s, 2s, 4s, max 30s). Fires a `connectionChange` callback (used by chatStore to update error state).
- **`sendMessage`**: Generates a `requestId` (ULID), sends `{ type: 'chat', requestId, content, context }` over WebSocket. Returns an `AsyncIterable<ChatEvent>` backed by an async generator that:
  - Listens for incoming WebSocket messages
  - Filters events matching the `requestId`
  - Yields each `ChatEvent` as it arrives
  - Completes on `{ type: 'done' }` or `{ type: 'error' }`
  - Handles `store_action` messages by calling `graphStore` methods and sending results back (see below)
- **`loadHistory`**: Sends `{ type: 'load_history', messages }` over WebSocket.
- **`abort`**: Sends `{ type: 'abort' }` over WebSocket.

**Store action handler** — when the WebSocket receives a `{ type: 'store_action', action, args, correlationId }` message (from the HTTP mutation path), the provider:
1. Calls the appropriate `graphStore` method (e.g., `graphStore.getState().addNode(...)`)
2. Calls `fileStore.getState().save()` if `fileStore.getState().fs` is not null (otherwise returns error)
3. Sends `{ type: 'store_action_result', correlationId, result }` back over WebSocket

This is the browser-side handler for the HTTP mutation API. It receives mutation requests relayed from the bridge and applies them through the stores.

Tests: WebSocket message serialization/deserialization, event filtering by requestId, AsyncIterable completion on done/error, store action handler calling correct graphStore methods, reconnection behavior (with mock WebSocket).

### `src/store/chatStore.ts`

Zustand store managing conversation state:

```typescript
interface ChatState {
  messages: ChatMessage[]
  isStreaming: boolean
  activeProviderId: string | null
  providers: Map<string, ChatProvider>
  error: string | null

  registerProvider(provider: ChatProvider): void
  setActiveProvider(id: string): void
  sendMessage(content: string): Promise<void>
  respondToPermission(id: string, allowed: boolean): void
  abort(): void
  clearHistory(): void
}
```

Key implementation details:
- **`sendMessage`**: Guards with `isStreaming` check. Gets active provider. Assembles `ProjectContext` from other stores via `getState()`:
  - `fileStore.getState()` → `projectName`, `projectPath` (from `projectDir`)
  - `navigationStore.getState()` → `currentScope` (from `currentCanvasId`)
  - Adds user message to `messages` array
  - Calls `provider.sendMessage(content, context)` → iterates `ChatEvent`s
  - Builds assistant message incrementally (text events append to content, all events stored in `events` array)
  - Sets `isStreaming = true` during iteration, `false` on completion
  - On `error` event: sets `error` state
- **`registerProvider`**: Adds provider to `providers` map. If no active provider, auto-selects it.
- **`respondToPermission`**: Forwards to active provider's WebSocket (sends `permission_response` ClientMessage).
- **`abort`**: Calls `provider.abort()`, sets `isStreaming = false`.

Follow the existing Zustand store patterns: `create<ChatState>()`, methods in the store definition, cross-store access via `getState()`.

Tests: message management (add user message, build assistant message from events), streaming state transitions, provider registration and selection, abort behavior, error handling, cross-store context assembly (mock other stores).
