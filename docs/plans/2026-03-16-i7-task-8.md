## Description

This task wires up the interactive features that connect entities to the rest of the UI: command palette search, edge highlighting, and the tab-switch API.

**Command palette `# ` prefix:**

The CommandPalette already supports prefix routing (e.g., `> ` for actions, `@ ` for nodes). Add a `# ` prefix for entities. When the user types `# Order`:
1. Call `listAllEntities(project)` and filter by query
2. Show results with entity name, description, and defining scope
3. On select: switch `uiStore.rightPanelMode` to `'entities'`, navigate to the entity's defining scope if different from current

Follow the existing provider pattern in CommandPalette — register a new provider with `category: 'ENTITIES'`, `prefix: '# '`.

**`highlightEdges` in canvasStore:**

New state and action:
```typescript
highlightedEdgeIds: string[]
highlightEdges: (edgeIds: string[]) => void
clearHighlight: () => void
```

Auto-clear triggers:
- Canvas click (anywhere, not just on a node/edge)
- Escape key
- Navigation change (scope change via diveIn/goUp)

The EntityPanel (Task 7) calls `highlightEdges()` when a user clicks an entity row, passing the IDs of all edges in the current scope that reference that entity.

**Edge highlight rendering:**

In `EdgeRenderer.tsx`, check if the edge's ID is in `highlightedEdgeIds`. If so, add `.edge-highlighted` CSS class. Style: brighter stroke color, subtle glow (e.g., `filter: drop-shadow(0 0 3px var(--edge-color))`). This is visually distinct from selection.

**Tab-switch API (§5, convenience):**

Add `detailPanelTab: 'properties' | 'notes' | 'codeRefs' | null` to `uiStore`. `NodeDetailPanel` subscribes and switches its active tab when this state changes. Setting to `null` means "use default." This is low-priority scaffolding for future use cases.

**Acceptance criteria:**
- `# Order` in command palette searches entities across scopes
- Selecting an entity navigates to its scope and opens EntityPanel
- Clicking an entity row in EntityPanel highlights related edges on canvas
- Highlighted edges have distinct visual style (glow)
- Highlights clear on click/Escape/navigation
- `setDetailPanelTab('notes')` switches NodeDetailPanel to Notes tab
