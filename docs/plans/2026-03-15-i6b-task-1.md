## Description

The onboarding wizard needs to auto-infer the project name from the directory the user selected. The `FileSystem` interface currently has no way to retrieve the directory name.

### Implementation

Add `getName(): string` to the `FileSystem` interface:

```typescript
export interface FileSystem {
  getName(): string;  // NEW ‚Äî returns the directory basename
  readFile(path: string): Promise<string>;
  // ... existing methods
}
```

Each implementation:

- **InMemoryFileSystem** ‚Äî add a `name` constructor parameter (default `'untitled'`). `getName()` returns it. Update existing test helpers that construct `InMemoryFileSystem` to optionally pass a name.
- **WebFileSystem** ‚Äî `getName()` returns `this.rootHandle.name` (the `FileSystemDirectoryHandle` already has a `.name` property).
- **NodeFileSystem** ‚Äî `getName()` returns `path.basename(this.rootPath)`. Import `basename` from `node:path`.
- **TauriFileSystem** ‚Äî `getName()` returns the last segment of the root path string.

### Testing

Test each implementation:
1. `InMemoryFileSystem` ‚Äî default name, custom name
2. `NodeFileSystem` ‚Äî extracts basename from absolute path
3. `WebFileSystem` ‚Äî returns handle.name (mock the handle)
