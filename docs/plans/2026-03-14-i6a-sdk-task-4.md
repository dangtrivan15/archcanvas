## Description

This task adds two user-facing controls to the ChatPanel header so the user can configure how the AI agent behaves.

### Permission Mode Selector

Add a small dropdown in the ChatPanel header (next to the provider selector) that lets the user choose the permission mode:

| Mode | Label | Description |
|---|---|---|
| `default` | Default | Ask for permission on write operations |
| `acceptEdits` | Auto-edit | Auto-approve file edits, ask for bash |
| `plan` | Plan only | Claude plans but doesn't execute |
| `dontAsk` | Strict | Only pre-approved tools, deny everything else |

**Note:** `bypassPermissions` is intentionally omitted from the UI to prevent accidental full-access mode. Power users can set it programmatically.

When changed, the store calls `webSocketProvider.sendSetPermissionMode(mode)`, which sends a `set_permission_mode` message over WebSocket. The vitePlugin (updated in T1) handles this by calling the bridge session's setPermissionMode method.

### Effort Toggle

Add a small segmented control or dropdown for effort level:

| Level | Label |
|---|---|
| `low` | Quick |
| `medium` | Medium |
| `high` | Thorough |
| `max` | Maximum |

Default: `high`. When changed, sends `set_effort` over WebSocket.

### chatStore.ts changes

1. **New state fields:** `permissionMode: string` (default: `'default'`), `effort: string` (default: `'high'`).

2. **`setPermissionMode(mode: string)`** — updates state, calls provider's `sendSetPermissionMode`.

3. **`setEffort(effort: string)`** — updates state, calls provider's `sendSetEffort`.

### ChatPanel.tsx changes

1. **Permission mode dropdown** — a `<select>` or custom dropdown in the header bar, next to the provider selector. Shows the 4 allowed modes.

2. **Effort selector** — a small segmented control or `<select>` showing Quick/Medium/Thorough/Maximum.

3. Both controls are disabled while streaming (can't change mid-request).

4. Keep the header compact — these are small controls, not full-width elements.
