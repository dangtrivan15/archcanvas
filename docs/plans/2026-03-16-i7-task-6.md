## Description

This task creates the entity resolver — a pure function layer that provides cross-scope entity lookup. Entities are defined per canvas in the YAML `entities:` array and referenced by name on edges. The resolver scans all loaded canvases to answer: "where is this entity defined?" and "where is it referenced?"

**Three functions:**

1. `findEntityUsages(project, entityName)` — scans all canvases' edges for entity name matches. Returns `EntityUsage[]` with canvas ID, display name, and matching edges. Useful for "show me everywhere Order is used."

2. `listAllEntities(project)` — collects all unique entity names across the project. For each, records which canvases define it (in their `entities:` array) and which canvases reference it (on edges). Returns `EntitySummary[]`. Useful for EntityPanel's main list view.

3. `getEntitiesForCanvas(project, canvasId)` — returns entities defined in a specific canvas scope. Simple map lookup + data extraction. Useful for scoped views.

**Entity matching is by name (string equality)**, consistent with how edges reference entities: `entities: [Order]` on an edge matches `{ name: Order }` in the entities array. No fuzzy matching, no ID-based lookup.

**Multi-canvas scanning**: The resolver iterates `project.canvases` (a Map). For `findEntityUsages` and `listAllEntities`, it also checks the root canvas (`project.root`). Performance is not a concern — projects have at most ~60 canvases, and scanning entities is O(canvases × edges).

**Test scenarios:**
- Single canvas with entities and edges → basic lookup
- Multi-canvas with same entity name in different scopes → cross-scope results
- Entity defined but not referenced → appears in `definedIn` but empty `referencedIn`
- Entity referenced on edge but not defined → appears in `referencedIn` but empty `definedIn`
- Empty project → empty results

**Acceptance criteria:**
- `findEntityUsages` returns correct canvases and edges for a given entity name
- `listAllEntities` returns all unique entities with correct definition/reference scopes
- `getEntitiesForCanvas` returns entities for a specific canvas
- Functions handle edge cases (empty project, missing entities, cross-scope references)
- All functions are pure (no side effects, no store access)
