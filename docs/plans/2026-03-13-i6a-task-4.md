## Description

This task creates the 5 React components that make up the chat panel UI. All components consume `chatStore` for state and follow the existing Tailwind dark theme patterns used by the detail panels and command palette.

### `src/components/panels/ChatPanel.tsx`

The main chat panel component. Layout:

```
┌──────────────────────────────┐
│ AI Chat    [Claude Code ▾] ✕ │  ← header
├──────────────────────────────┤
│ (scrollable message list)    │  ← body
│   ChatMessage (user)         │
│   ChatMessage (assistant)    │
│     ChatToolCall (collapsed) │
│     ChatPermissionCard       │
├──────────────────────────────┤
│ [input box]          [Send] │  ← footer
└──────────────────────────────┘
```

Implementation:
- Header: title, `ChatProviderSelector`, close button (dispatches `archcanvas:toggle-chat`)
- Body: scrollable `div` that auto-scrolls to bottom on new messages. Maps `chatStore.messages` to `ChatMessage` components.
- Footer: text input (auto-focus when panel opens), send button. Submit on Enter (Shift+Enter for newline). Disabled when `isStreaming` or provider not `available`.
- Shows connection error banner when `chatStore.error` is set.
- Shows streaming indicator (pulsing dot) when `isStreaming`.

### `src/components/panels/ChatMessage.tsx`

Single message bubble:
- User messages: right-aligned, darker background (`bg-surface-2` or similar)
- Assistant messages: left-aligned, lighter background
- Assistant messages render `events` array: text content is displayed as formatted text, `tool_call`/`tool_result` events render as `ChatToolCall`, `permission_request` events render as `ChatPermissionCard`, `thinking` events render as dimmed collapsible section
- Timestamp displayed subtly

### `src/components/panels/ChatToolCall.tsx`

Collapsible tool call display:
- Header (always visible): tool name + command summary (e.g., "Bash: archcanvas add-node...")
- Collapsed by default, click to expand
- Expanded: shows full args as formatted JSON, and result (if `tool_result` event exists)
- Error results highlighted with red accent

### `src/components/panels/ChatPermissionCard.tsx`

Permission request card:
- Shows the tool name and full command
- Two buttons: "Approve" (green accent) and "Deny" (red accent)
- Calls `chatStore.respondToPermission(id, allowed)` on click
- Disabled after response (shows "Approved" or "Denied" state)
- Only interactive when the permission hasn't been responded to yet

### `src/components/panels/ChatProviderSelector.tsx`

Backend selector dropdown:
- Lists all registered providers from `chatStore.providers`
- Shows provider `displayName` and availability indicator (green dot if `available`, gray if not)
- Currently only one option (Claude Code) — styled as a dropdown that's ready for more
- On select: calls `chatStore.setActiveProvider(id)`

### Styling Notes

Follow existing dark theme patterns:
- Use `bg-[var(--color-surface-*)]` CSS variables from the existing theme
- Text colors: `text-[var(--color-text)]`, `text-[var(--color-text-muted)]`
- Borders: `border-[var(--color-border)]`
- Reference `CommandPalette.tsx` and `NodeDetailPanel.tsx` for consistent styling
