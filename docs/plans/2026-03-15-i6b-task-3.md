### Part 2: `OnboardingWizard.tsx`

Top-level wizard component. Manages step routing via local state:

```typescript
type WizardStep = 'init-method' | 'ai-survey';

export function OnboardingWizard() {
  const [step, setStep] = useState<WizardStep>('init-method');

  return (
    <div className="flex h-screen flex-col items-center justify-center bg-background text-foreground">
      {step === 'init-method' && (
        <InitMethodStep
          onBlankCanvas={() => useFileStore.getState().completeOnboarding('blank')}
          onAiAnalyze={() => setStep('ai-survey')}
        />
      )}
      {step === 'ai-survey' && (
        <AiSurveyStep
          onBack={() => setStep('init-method')}
          onStart={(survey) => useFileStore.getState().completeOnboarding('ai', survey)}
        />
      )}
    </div>
  );
}
```

### Part 3: `InitMethodStep.tsx`

Step 1 — two clickable cards (AI Analyze / Blank Canvas). Visual style matches `ProjectGate`.

Key details:
- **AI availability**: subscribe to `useChatStore` providers map. Check if any provider has `available === true`. Show "Requires AI connection" muted hint when unavailable.
- The availability hint is informational only — the card is always clickable (the user proceeds to the survey, and the "Start" button in Step 2 is the gated action).
- **Blank Canvas card** calls `onBlankCanvas()` directly (no next step).
- **AI Analyze card** calls `onAiAnalyze()` to advance to Step 2.

### Part 4: `AiSurveyStep.tsx`

Step 2 — survey form for AI initialization context.

Fields (matching `SurveyData` type from Task 2):
- **Description** — `<textarea>`, required. Placeholder: "Describe what this project does..."
- **Tech stack** — chip selector. Array of preset options. Multi-select toggle. Visually: small rounded chips, filled when selected.
- **Exploration depth** — `<select>` dropdown: "Full" (default), "Top-level only", "Custom". When "Custom" is selected, show a numeric input for max depth.
- **Focus directories** — `<input type="text">`. Placeholder: "e.g., src/, services/ (leave empty for all)"

Buttons:
- "Back" → calls `onBack()`
- "Start" → calls `onStart(surveyData)`. **Disabled when AI provider is not available.** Read availability reactively from `useChatStore`.

When AI is unavailable, show an amber banner: "AI is not connected. The analysis will start once a connection is established."
