## Description

This task creates Playwright E2E tests that verify the full AI chat experience in the browser. These tests exercise the integrated system — UI toggle, chat panel rendering, and interaction flows — unlike the unit/integration tests in T1-T6 which test components in isolation.

### Important Note on Test Scope

E2E tests run against `vite preview` (production build). The Vite AI bridge plugin only runs in `vite dev` mode. Therefore, these E2E tests focus on **UI interactions and rendering** — they cannot test live AI conversations. The full pipeline (mock SDK → bridge → WebSocket → provider → store → UI) is covered by the integration tests in T2 and T3.

### `test/e2e/ai-chat.spec.ts`

**Chat panel toggle:**
1. Verify AI Chat button exists in left toolbar with correct aria-label
2. Click AI Chat button → right panel shows chat panel (verify header text "AI Chat")
3. Click again → right panel returns to details mode (verify "Select a node" placeholder)
4. `Cmd+Shift+I` → chat panel opens
5. `Cmd+Shift+I` again → chat panel closes

**Chat panel layout:**
1. Open chat panel → verify header, provider selector, close button visible
2. Verify provider selector shows "Claude Code"
3. Verify input box is present with placeholder text
4. Verify send button is present

**Chat toggle + node selection interaction:**
1. Open chat panel
2. Close chat panel (returns to details mode)
3. Click a node on canvas → verify NodeDetailPanel appears (no regression)
4. Open chat panel while node is selected → verify chat panel replaces details
5. Close chat panel → verify NodeDetailPanel returns for the selected node

**Panel collapse interaction:**
1. Collapse right panel (via drag or existing toggle)
2. Click AI Chat toolbar button → verify right panel expands AND shows chat
3. Close chat → verify right panel shows details mode (remains expanded)

**Chat button visual state:**
1. When chat is closed → verify toolbar button is in default state
2. When chat is open → verify toolbar button has active/highlighted state
3. Toggle back → verify button returns to default state

### Acceptance Criteria

- All E2E tests pass against production build (`vite preview`)
- No regressions in existing E2E tests (`app-shell`, `canvas-operations`, `file-operations`)
- Tests use existing Playwright patterns (slot guard, port 4173)
- Chat panel interactions are smooth — no flicker, correct panel switching
