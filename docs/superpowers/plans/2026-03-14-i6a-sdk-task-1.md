# I6a SDK Improvements — Task 1: Types & Bridge Core Options

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Expand the type system and bridge plumbing to support all Agent SDK capabilities — missing tools, safety guards, richer permission context, streaming, configurable effort/mode, and auto-approve hooks.

**Architecture:** The bridge wraps the SDK `query()` function. We expand the `SDKQueryFn` type to accept the full SDK options, expand `ChatEvent`/`ClientMessage` unions for new event types, and wire new session-level settings (permissionMode, effort) through the bridge and WebSocket transport.

**Tech Stack:** TypeScript, Claude Agent SDK, Vite plugin, WebSocket

respondToPermission(

      updatedPermissions: options?.updatedPermissions,
