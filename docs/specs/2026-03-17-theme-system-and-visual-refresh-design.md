# Theme System & Visual Refresh

> **Date**: 2026-03-17 | **Phase**: Spec
> **Scope**: Configurable color palettes (light/dark), text size presets, settings UI, font upgrade, visual polish

## Problem

ArchCanvas v2 has a single hardcoded dark theme (zinc-based), system fonts with no size control, and no settings UI. The app looks functional but visually unrefined:

1. **No theme switching** — dark-only, hardcoded CSS variables in `@theme`, no light mode
2. **No font identity** — `system-ui` fallback chain, no monospace font for code/technical content
3. **No user preferences** — no way to configure appearance (palette, mode, text size)
4. **Inconsistent styling** — mix of hardcoded hex colors in `nodeShapes.css`, `EdgeRenderer.css`, and inline component styles alongside CSS variables
5. **Ugly resize handles** — `react-resizable-panels` default grip icon (`⠿`) is visually noisy
6. **Unstyled ReactFlow controls** — default zoom/fit buttons don't match app aesthetic

---

## §1 — Design Language (Fixed)

The design language is ArchCanvas's visual identity — it stays constant regardless of palette or mode. It is **not configurable**.

### Typography
- **Sans-serif**: Inter (weights 400, 500, 600)
- **Monospace**: Monaspace Argon (weights 400, 500) — for code refs, property values, technical identifiers
- **Fallback chains**: `'Inter', system-ui, -apple-system, sans-serif` and `'Monaspace Argon', ui-monospace, monospace`
- **Font loading**: Self-hosted in `public/fonts/`, no external CDN. Works offline in Tauri.

### Spatial Design
- **Border-radius**: Update `--radius` to `0.625rem` (10px at medium text size). Shadcn/ui components derive `rounded-lg` (`--radius`), `rounded-md` (`calc(--radius - 2px)`), `rounded-sm` (`calc(--radius - 4px)`) from this single variable — no additional radius tokens needed.
- **Shadows**: Subtle, layered — `0 1px 3px rgba(0,0,0,0.04)` for light, `0 1px 3px rgba(0,0,0,0.3)` for dark
- **Spacing rhythm**: 4px base unit, consistent padding/margins across panels
- **Border widths**: 1px default, 2px for selected state

### Interaction
- **Resize handles**: Invisible at rest, faint accent-colored line appears on hover. 5px hit zone. Cursor changes to `col-resize`/`row-resize`.
- **ReactFlow controls**: Restyled to match current palette — rounded corners, themed backgrounds, consistent with toolbar buttons.

## §2 — Theme Token Architecture

### Semantic Tokens

Every palette must provide these tokens for both light and dark modes. These map directly to existing `--color-*` CSS custom properties.

#### Relationship with Tailwind 4's `@theme` Block

The current `@theme` block in `index.css` defines CSS custom properties at build time (e.g., `--color-background: #09090b`). Tailwind uses these to generate utility classes like `bg-background`, `text-foreground`, etc.

At runtime, `applyTheme.ts` overrides these values via `document.documentElement.style.setProperty('--color-background', newValue)`. This works because inline styles on `<html>` have higher specificity than `:root` declarations generated by `@theme`.

The `@theme` block retains the **ArchCanvas Default dark-mode values** as compile-time defaults. This ensures:
- Tailwind can generate all utility classes at build time
- If JS fails to load, the app falls back to a working dark theme
- New canvas tokens (`node-bg`, `edge-sync`, etc.) must also be added to `@theme` with default values

#### UI Tokens

| Token | Purpose |
|-------|---------|

| `foreground` | Default text color |

| `accent` | Accent/highlight background |
| `accent-foreground` | Text on accent elements |

  id: string;          // 'archcanvas' | 'rose-pine' | 'catppuccin'

### Palette File Structure

```
src/core/theme/
├── types.ts              # ThemeTokens, ThemePalette
├── applyTheme.ts         # Applies tokens as CSS custom properties
├── palettes/
│   ├── index.ts          # Barrel export, palette registry array
│   ├── archcanvas.ts     # ArchCanvas Default palette
│   ├── rosePine.ts       # Rose Pine palette
│   └── catppuccin.ts     # Catppuccin palette
```

Each palette file exports a `ThemePalette` object with `light` and `dark` token sets. All color values are hex strings (e.g., `'#faf4ed'`), matching the format used in the existing `@theme` block.

#### Example (partial — Rose Pine Dawn light)

```typescript
export const rosePine: ThemePalette = {
  id: 'rose-pine',
  name: 'Rose Pine',
  light: {
    background: '#faf4ed',
    foreground: '#575279',
    card: '#fffaf3',
    cardForeground: '#575279',
    muted: '#f2e9e1',
    mutedForeground: '#9893a5',
    primary: '#907aa9',
    // ... all tokens
  },
  dark: {
    background: '#191724',
    foreground: '#e0def4',
    // ... all tokens
  },

```

#### Token Name Conversion

TypeScript camelCase token names convert to CSS kebab-case custom properties: `cardForeground` → `--color-card-foreground`, `nodeBg` → `--color-node-bg`, `edgeSync` → `--color-edge-sync`. The conversion splits on uppercase boundaries.

## §3 — ThemeStore

### State & Actions

```typescript
// src/store/themeStore.ts

interface ThemeState {
  palette: string;                          // palette id
  mode: 'light' | 'dark' | 'system';
  textSize: 'small' | 'medium' | 'large';
}

interface ThemeActions {
  setPalette: (palette: string) => void;
  setMode: (mode: 'light' | 'dark' | 'system') => void;
  setTextSize: (textSize: 'small' | 'medium' | 'large') => void;
}
```

`getResolvedMode` is a **derived getter** (not a state-mutating action) — exposed as a function on the store for convenience. It reads `mode` and queries `prefers-color-scheme` when mode is `'system'`.

### Defaults

| Setting | Default |
|---------|---------|
| `palette` | `'archcanvas'` |
| `mode` | `'system'` |
| `textSize` | `'medium'` |

### Persistence

- **Storage key**: `archcanvas:theme`
- **Format**: JSON `{ palette, mode, textSize }`
- **Mechanism**: Read/write `localStorage` on every change

### Text Size Mapping

| Preset | `<html>` font-size | Effect |
|--------|-------------------|--------|
| `small` | 13px | Compact UI |
| `medium` | 15px | Default |
| `large` | 17px | Accessibility-friendly |

Applied as `font-size` on `document.documentElement`. Components using `rem` units scale automatically.

**Note on canvas text**: Node labels and edge labels in `nodeShapes.css` / `EdgeRenderer.css` currently use hardcoded `px` values (e.g., `12px`, `10px`). These will **not** scale with the text size preset — this is intentional, since canvas content has its own zoom level via ReactFlow. The text size preset controls only the UI chrome (panels, menus, toolbar, status bar).

### Theme Application (`applyTheme.ts`)

On any state change:

1. Resolve the effective mode (`system` → query `prefers-color-scheme`)
2. Look up the palette by id from the palette registry
3. Select the `light` or `dark` token set
4. Apply each token as a CSS custom property on `<html>`: `--color-{kebab-case-token}: {value}`
5. Apply `font-size` on `<html>` based on `textSize`

### System Mode Listener

When `mode === 'system'`, subscribe to `window.matchMedia('(prefers-color-scheme: dark)')` change events. On change, re-apply the theme. Unsubscribe when mode changes away from `system`.

### Flash Prevention

To avoid a flash of wrong theme on page load:

1. Add a synchronous `<script>` in `<head>` of `index.html` (after `<meta>` tags, before stylesheets)
2. The script reads `localStorage` for `archcanvas:theme`
3. Applies `font-size` on `<html>` immediately
4. The script also sets all `--color-*` CSS custom properties — it contains a **minimal inline copy** of each palette's token values (a compact JSON map). This is ~1-2KB total for 3 palettes and avoids an extra network request.
5. When React mounts, `themeStore` initializes from the same localStorage data — no visual change

If the localStorage data is missing or corrupt, the script falls back to ArchCanvas Default palette with `mode: 'system'` (querying `prefers-color-scheme`).

## §5 — Visual Polish

### 5a — Font Loading

Add `@font-face` declarations in `index.css`:

- Inter: 400 (regular), 500 (medium), 600 (semi-bold) — `woff2` format
- Monaspace Argon: 400 (regular), 500 (medium) — `woff2` format
- Files stored in `public/fonts/inter/` and `public/fonts/monaspace-argon/`
- Update `body` font-family to `'Inter', system-ui, -apple-system, sans-serif`
- Define `--font-family-mono` in `@theme` for `'Monaspace Argon', ui-monospace, monospace` — this enables Tailwind's `font-mono` utility class

### 5b — Node Shapes (`nodeShapes.css`)

Migrate all hardcoded hex colors to CSS custom properties:

| Current | Replacement |
|---------|-------------|
| `background: white` / `#fff` | `var(--color-node-bg)` |
| `border-color: #e5e7eb` | `var(--color-node-border)` |
| `box-shadow: ...` with hardcoded color | `var(--color-node-shadow)` |
| `border-color: #3b82f6` (selected) | `var(--color-node-selected-border)` |
| `box-shadow: 0 0 0 2px ...` (selected) | `var(--color-node-selected-ring)` |
| Ref node bg/border | `var(--color-node-ref-bg)`, `var(--color-node-ref-border)` |

### 5c — Edge Protocols (`EdgeRenderer.css`)

Migrate hardcoded edge colors:

| Current | Replacement |
|---------|-------------|
| `stroke: #374151` (sync) | `var(--color-edge-sync)` |
| `stroke: #6366f1` (async) | `var(--color-edge-async)` |
| `stroke: #9ca3af` (default) | `var(--color-edge-default)` |

Additionally, migrate hardcoded colors for other edge-related elements:

| Element | Current vars/hardcoded | Replacement |
|---------|----------------------|-------------|
| `.edge-label` bg/border/text | `#ffffff`, `#e5e7eb`, `#374151` | `var(--color-card)`, `var(--color-border)`, `var(--color-foreground)` |
| `.entity-pill` bg/border/text | `#f3f4f6`, `#d1d5db`, `#6b7280` | `var(--color-muted)`, `var(--color-border)`, `var(--color-muted-foreground)` |
| `.ghost-node` vars | `--color-border-muted`, `--color-bg-muted`, `--color-text-muted` (non-existent, always fallback) | `var(--color-border)`, `var(--color-muted)`, `var(--color-muted-foreground)` |

**Warning/unknown-type colors**: `.arch-node.unknown-type` (amber `#f59e0b`/`#fffbeb`) and `.arch-node-warning` (amber `#d97706`/`#fef3c7`/`#fcd34d`) remain **hardcoded**. These are semantic warning indicators that should stay consistent across all palettes — similar to how `destructive` conveys a fixed meaning.

Keep animated dash on async edges — only colors change.

### 5d — Resize Handles

Replace `react-resizable-panels` default grip with custom handle:

- **At rest**: Invisible — no visual indicator
- **On hover**: Faint accent-colored line (2px, 30% opacity) fades in over 150ms
- **Hit zone**: 5px wide for easy grabbing
- **Cursor**: `col-resize` (vertical panels) or `row-resize` (horizontal panels)

Implementation: Custom `handle` component passed to `<PanelResizeHandle>` — a `div` with the accent line styled via CSS transition.

### 5e — ReactFlow Controls

Override default `<Controls>` styling to match the current palette:

- Background: `var(--color-card)`
- Border: `var(--color-border)`
- Icons: `var(--color-muted-foreground)`
- Hover: `var(--color-accent)` background
- Border-radius: 6px, consistent with toolbar buttons

### 5f — Hardcoded Color Audit

Scan all components for hardcoded hex colors and replace with appropriate CSS variable references. Key areas:

- Inline styles in components using literal `#hex` values
- ReactFlow dark theme overrides in `index.css`
- Status bar, breadcrumb, context menu colors

## §7 — Persistence Note

Preferences are stored in `localStorage` (key: `archcanvas:theme`). This is the simplest approach and works reliably in both browser and Tauri environments.

**Future upgrade path**: If cross-device or cross-browser sync is needed, file-based persistence (global `~/.archcanvas/preferences.yaml` or per-project `.archcanvas/theme.yaml`) can be layered on without changing the store architecture — the store would simply read/write to a file instead of (or in addition to) localStorage.

## §8 — Edge Cases

| Scenario | Behavior |
|----------|----------|
| `localStorage` contains unknown palette ID (e.g., palette removed in future version) | Fall back to `'archcanvas'` default palette |
| `localStorage` data is corrupt / unparseable JSON | Fall back to all defaults: `{ palette: 'archcanvas', mode: 'system', textSize: 'medium' }` |
| `prefers-color-scheme` not supported (older browsers) | When mode is `'system'`, default to `'light'` |
| Font files fail to load (network/filesystem issue) | Fallback chain handles it: `system-ui` for sans, `ui-monospace` for mono. No broken UI. |
| `window`/`document` unavailable (SSR/testing) | `applyTheme.ts` guards all DOM access with `typeof document !== 'undefined'`. Store state works without DOM. |
| User switches OS dark/light mode while app is open | `matchMedia` listener fires, re-applies theme immediately (only when mode is `'system'`) |
