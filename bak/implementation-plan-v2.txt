# ArchCanvas — Full Implementation Plan v2
# Binary Core + Text API + MCP Server Architecture

## ============================================================
## 1. PROJECT OVERVIEW
## ============================================================

ArchCanvas is a visual architecture design tool where software systems
are represented as typed, composable, code-bound graph nodes on an
interactive canvas.

### Core Principles
- The `.archc` binary file is the SINGLE source of truth
- ALL access goes through three APIs: Render, Text, Export
- Nobody reads binary directly — ever
- Community nodedefs are open YAML; architecture data is proprietary binary
- AI and humans interact through the same Text API
- The canvas and codebase evolve together

### MVP Scope (Phase 1-6)
- Desktop web app (runs in browser, later wraps in Tauri)
- Binary core engine with protobuf storage
- Text API with ai/human/structured output formats
- Interactive canvas with typed nodes and edges
- Fractal zoom (navigate into child nodes)
- 15 built-in nodedefs (YAML, open)
- Notes panel, properties panel, code ref panel
- AI chat panel scoped to selected node context
- MCP server for AI agent integration
- Markdown summary sidecar (auto-generated on save)
- Export: Mermaid, PNG, SVG

### NOT in MVP
- IDE extensions (VS Code / JetBrains) — Phase 7+
- Real-time collaboration — Phase 8+
- Community nodedef registry server — Phase 9+
- iPad native shell — Phase 10+
- Multi-file $ref splitting — Phase 7+
- CLI tool — Phase 7+

## ============================================================
## 2. TECH STACK
## ============================================================

### Core Application
- Language:           TypeScript (strict mode)
- Build:              Vite 6
- UI Framework:       React 19
- Canvas:             React Flow (@xyflow/react) v12
- Layout Engine:      elkjs (ELK compiled to JS)
- Styling:            Tailwind CSS 4 + shadcn/ui
- State:              Zustand (global app state)
- Icons:              Lucide React

### Binary Core
- Serialization:      protobufjs (Protocol Buffers for JS)
                      — compiles .proto schemas to TS classes
                      — handles encode/decode of .archc files
- Schema Validation:  Zod (validates data at API boundaries)
- File Handling:      File System Access API (Chrome)
                      + fallback to File/Blob download

### NodeDef Registry
- Format:             YAML
- Parser:             yaml (npm "yaml" package, YAML 1.2)
- Validation:         Zod schemas for nodedef structure

### AI Integration
- LLM:               Anthropic Claude API (@anthropic-ai/sdk)
- MCP Server:         @modelcontextprotocol/sdk
- Protocol:           MCP over stdio (for Claude Code)
                      + MCP over SSE (for web-based agents)

### Testing
- Unit:               Vitest
- Component:          React Testing Library
- E2E:                Playwright
- Proto:              protobufjs built-in verification

### Dev Tools
- Linting:            ESLint + Prettier
- ID Generation:      ulid
- Export:              html-to-image (PNG/SVG), custom Mermaid generator

## 4. PROTOCOL BUFFER SCHEMA (proto/archcanvas.proto)
## ============================================================

```protobuf
syntax = "proto3";
package archcanvas;

// ── File Container ─────────────────────────────────

message ArchCanvasFile {
  // Magic: first 8 bytes of serialized file = "ARCHC\x00" + uint16 version
  // This is handled in codec.ts, not in protobuf itself

  FileHeader header = 1;
  Architecture architecture = 2;
  CanvasState canvas_state = 3;
  AIState ai_state = 4;
  UndoHistory undo_history = 5;
}

message FileHeader {
  uint32 format_version = 1;     // current: 1
  string tool_version = 2;       // e.g. "0.1.0"
  int64 created_at_ms = 3;       // unix millis
  int64 modified_at_ms = 4;      // unix millis
  string checksum_sha256 = 5;    // SHA-256 of Architecture bytes
}

// ── Architecture Graph ─────────────────────────────

message Architecture {
  string name = 1;
  string description = 2;
  repeated string owners = 3;
  repeated Node nodes = 4;
  repeated Edge edges = 5;
}

message Node {
  string id = 1;
  string type = 2;               // nodedef ref: "core/compute/service"
  string display_name = 3;
  map<string, Value> args = 4;
  repeated CodeRef code_refs = 5;
  repeated Note notes = 6;
  map<string, string> properties = 7;
  Position position = 8;
  repeated Node children = 9;    // recursive for fractal zoom
  string ref_source = 10;        // only for type="$ref"
}

message Edge {
  string id = 1;
  string from_node = 2;
  string to_node = 3;
  string from_port = 4;
  string to_port = 5;
  EdgeType type = 6;
  string label = 7;
  map<string, Value> properties = 8;
  repeated Note notes = 9;
}

enum EdgeType {
  EDGE_SYNC = 0;
  EDGE_ASYNC = 1;
  EDGE_DATA_FLOW = 2;
}

message Note {
  string id = 1;
  string author = 2;
  int64 timestamp_ms = 3;
  string content = 4;            // Markdown
  repeated string tags = 5;
  NoteStatus status = 6;
  string suggestion_type = 7;    // "warning", "improvement", "note"
}

enum NoteStatus {
  NOTE_NONE = 0;
  NOTE_PENDING = 1;
  NOTE_ACCEPTED = 2;
  NOTE_DISMISSED = 3;
}

message CodeRef {
  string path = 1;
  CodeRefRole role = 2;
}

enum CodeRefRole {
  ROLE_SOURCE = 0;
  ROLE_API_SPEC = 1;
  ROLE_SCHEMA = 2;
  ROLE_DEPLOYMENT = 3;
  ROLE_CONFIG = 4;
  ROLE_TEST = 5;
}

message Position {
  double x = 1;
  double y = 2;
  double width = 3;
  double height = 4;
  string color = 5;
}

// ── Canvas UI State ────────────────────────────────

message CanvasState {
  double viewport_x = 1;
  double viewport_y = 2;
  double viewport_zoom = 3;
  repeated string selected_node_ids = 4;
  repeated string navigation_path = 5;
  PanelLayout panel_layout = 6;
}

message PanelLayout {
  bool right_panel_open = 1;
  string right_panel_tab = 2;   // "notes", "properties", "coderef", "ai"
  double right_panel_width = 3;
}

// ── AI State ───────────────────────────────────────

message AIState {
  repeated AIConversation conversations = 1;
}

message AIConversation {
  string id = 1;
  string scoped_to_node_id = 2;
  repeated AIMessage messages = 3;
  int64 created_at_ms = 4;
}

message AIMessage {
  string id = 1;
  string role = 2;               // "user" or "assistant"
  string content = 3;
  int64 timestamp_ms = 4;
  repeated AISuggestion suggestions = 5;
}

message AISuggestion {
  string id = 1;
  string target_node_id = 2;
  string target_edge_id = 3;
  string suggestion_type = 4;    // "add-note", "warning", "improvement"
  string content = 5;
  NoteStatus status = 6;
}

// ── Undo History ───────────────────────────────────

message UndoHistory {
  repeated UndoEntry entries = 1;
  int32 current_index = 2;      // pointer into entries
  int32 max_entries = 3;         // cap, default 100
}

message UndoEntry {
  string description = 1;
  int64 timestamp_ms = 2;
  bytes architecture_snapshot = 3; // serialized Architecture message
}

// ── Generic Value ──────────────────────────────────

message Value {
  oneof kind {
    string string_value = 1;
    double number_value = 2;
    bool bool_value = 3;
    StringList list_value = 4;
  }
}

message StringList {
  repeated string values = 1;
}
```

    // 2. Deserialize entry at currentIndex
    // 3. Return graph state

  async *chat(
    messages: Array<{ role: "user" | "assistant"; content: string }>,
    systemPrompt: string,
    contextMessage: string
  ): AsyncIterable<string> {
    // Prepend context as first user message
    const fullMessages = [
      { role: "user" as const, content: contextMessage },
      { role: "assistant" as const, content: "I've reviewed the architecture. How can I help?" },
      ...messages
    ];

    const stream = await this.anthropic.messages.stream({
      model: "claude-sonnet-4-20250514",
      max_tokens: 4096,
      system: systemPrompt,
      messages: fullMessages
    });

    for await (const event of stream) {
      if (event.type === "content_block_delta" && event.delta.type === "text_delta") {
        yield event.delta.text;
      }
    }
  }
}

export function registerTools(server: McpServer, textApi: TextAPI) {

        nodeId,

        includeNotes: true,
        includeAIContext: true,
        includeCodeRefs: true
      });

    }
  );

  server.tool(

  );

      targetId: z.string().describe("Node or edge ID"),

        targetId,
        targetType,

      args: z.record(z.unknown()).optional(),
      properties: z.record(z.string()).optional(),

#### Step 6.4: Auto-Layout (ELK Integration)

```typescript
// src/core/layout/elkLayout.ts
import ELK from "elkjs/lib/elk.bundled";

export async function computeLayout(
  nodes: ArchNode[],
  edges: ArchEdge[],
  options: {
    direction: "DOWN" | "RIGHT";
    spacing: number;
    includeChildren: boolean;
  }
): Promise<Map<string, Position>> {
  const elk = new ELK();

  // Convert to ELK format
  const elkGraph = {
    id: "root",
    layoutOptions: {
      "algorithm": "layered",
      "elk.direction": options.direction,
      "spacing.nodeNode": String(options.spacing),
      "hierarchyHandling": options.includeChildren
        ? "INCLUDE_CHILDREN" : "SEPARATE_CHILDREN"
    },
    children: nodes.map(n => ({
      id: n.id,
      width: n.position.width || 280,
      height: n.position.height || 160,
      children: options.includeChildren
        ? n.children.map(c => ({ id: c.id, width: 240, height: 120 }))
        : undefined
    })),
    edges: edges.map(e => ({
      id: e.id,
      sources: [e.fromNode],
      targets: [e.toNode]
    }))
  };

  const result = await elk.layout(elkGraph);

  // Extract positions
  const positions = new Map<string, Position>();
  function extract(node: any) {
    positions.set(node.id, {
      x: node.x ?? 0,
      y: node.y ?? 0,
      width: node.width,
      height: node.height
    });
    node.children?.forEach(extract);
  }
  result.children?.forEach(extract);

  return positions;
}
```
