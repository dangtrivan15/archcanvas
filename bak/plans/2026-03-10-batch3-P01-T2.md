## Description

Migrate 7 dialogs that follow the standard pattern: boolean flag + info object in
uiStore, rendered inline in App.tsx.

### Per-Dialog Migration Steps

For each dialog:

1. **Move file** from `src/components/shared/` to `src/dialogs/`
2. **Add self-registration** at module level:
   ```typescript
   import { registerDialog } from './registry';
   registerDialog({ id: 'delete', component: DeleteConfirmationDialog });
   ```
3. **Update props interface** — The dialog receives its props from DialogHost via
   `dialogProps.get(id)` instead of reading from uiStore selectors. Change:
   ```typescript
   // Before: dialog reads its own open state
   const info = useUIStore(s => s.deleteDialogInfo);
   const close = useUIStore(s => s.closeDeleteDialog);

   // After: dialog receives props, uses generic close
   interface DeleteConfirmationDialogProps {
     nodeId: string; nodeName: string; edgeCount: number; childCount: number;
     nodeIds?: string[]; nodeCount?: number;
   }
   // Props passed in from DialogHost, close via closeDialog('delete')
   ```
4. **Update imports** in the dialog file (path changes from `@/components/shared/` to `@/dialogs/`)
5. **Ensure registration import** — Add `import '@/dialogs/DeleteConfirmationDialog'`
   to `src/dialogs/index.ts` so registration runs at import time.

### Dialog-Specific Notes

- **DeleteConfirmationDialog** (232 lines): Also imports from graphStore for deletion.
  Keep those imports — only change the open/close/info pattern.
- **ConnectionTypeDialog** (227 lines): Imports from graphStore for edge creation.
  Keep those imports.
- **ConflictDialog** (151 lines): Has `onReload` and `onSaveAsCopy` callbacks in info.
  These become props passed through the registry.
- **EmptyProjectDialog** (162 lines): Has `onUseAI` and `onQuickScan` callbacks.
  Same pattern — callbacks are in the info/props object.
