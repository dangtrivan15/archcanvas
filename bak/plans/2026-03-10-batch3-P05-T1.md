## Description

Merge two separate navigation systems (fractal zoom within a file + cross-file
navigation) into a single unified store with a backwards-compatible facade.

### Unified State Model

```typescript
interface NavigationState {
  // ── Core State ──
  breadcrumb: BreadcrumbEntry[];      // Full path from root to current view
  viewportStack: ViewportEntry[];     // Saved viewports per depth level
  parentEdgeIndicators: ParentEdgeIndicator[];  // Portal indicators (from nestedCanvasStore)

  // ── Actions ──
  diveIntoNode: (nodeId: string) => void;           // Fractal zoom (was zoomIn)
  diveIntoFile: (filePath: string, graph: ArchGraph, containerNodeId?: string, containerColor?: string) => void;  // Cross-file (was pushFile)
  goUp: () => void;                                  // Pop one level (was zoomOut/popFile)
  goToRoot: () => void;                              // Reset to root (was zoomToRoot/popToRoot)
  goToDepth: (index: number) => void;                // Jump to breadcrumb level (was zoomToLevel)
  reset: () => void;                                 // Clear all navigation state

  // ── Computed ──
  currentDepth: number;               // breadcrumb.length
  currentNodePath: string[];          // Node IDs only (for renderApi compatibility)
  isAtRoot: boolean;                  // breadcrumb.length === 0
  fileDepth: number;                  // Count of 'file' entries in breadcrumb
  nodeDepth: number;                  // Count of 'node' entries in breadcrumb
}

interface BreadcrumbEntry {
  type: 'node' | 'file';
  id: string;                         // nodeId or filePath
  displayName: string;
  viewport: ViewportEntry;            // Saved viewport at this level
  graph?: ArchGraph;                  // For file entries: the saved parent graph
  containerColor?: string;            // For file entries: container node color
}

interface ViewportEntry {
  x: number;
  y: number;
  zoom: number;
}
```

### Backwards-Compatible API

The unified store MUST export these aliases so existing consumers don't break:

```typescript
// Old navigationStore API → aliases
path: string[]                // = currentNodePath (derived from breadcrumb)
zoomIn: (nodeId) => void      // = diveIntoNode
zoomOut: () => void           // = goUp
zoomToRoot: () => void        // = goToRoot
zoomToLevel: (path) => void   // = goToDepth (converts path to index)
```

These are Zustand computed selectors, not state — derived from `breadcrumb` on read.

### nestedCanvasStore Facade

Convert `src/store/nestedCanvasStore.ts` into a thin facade:

```typescript
// Re-export everything from unified navigationStore
export const useNestedCanvasStore = useNavigationStore;
// Or, if the shape differs, create a selector-based wrapper
```

Key re-exports needed:
- `fileStack` → derived from `breadcrumb.filter(e => e.type === 'file')`
- `parentEdgeIndicators` → directly from unified store
- `pushFile` → `diveIntoFile`
- `popFile` → `goUp` (when current is file entry)
- `popToRoot` → `goToRoot`
- `getDepth` → `fileDepth`

### Viewport Save/Restore Logic

When `diveIntoNode(nodeId)`:
1. Save current viewport to `viewportStack[currentDepth]`
2. Push BreadcrumbEntry with type 'node'
3. Caller (useCanvasNavigation or useContainerDiveIn) handles the viewport transition

When `diveIntoFile(filePath, graph, ...)`:
1. Save current viewport + graph to BreadcrumbEntry
2. Push BreadcrumbEntry with type 'file'
3. Call `graphStore._setGraph(graph)` to swap the active graph
4. Call `canvasStore.requestFitView()` for initial layout

When `goUp()`:
1. Pop top BreadcrumbEntry
2. If popped entry was 'file': restore parent graph via `graphStore._setGraph()`
3. Restore viewport from `viewportStack[newDepth]`
4. Update `parentEdgeIndicators` based on new top-of-stack

### Key Decisions

- `BreadcrumbEntry.graph` stores complete graph snapshots (same as current FileStackEntry).
  Not ideal for memory, but matches existing behavior and is correct.
- `parentEdgeIndicators` computation (`buildParentEdgeIndicators`) moves into the
  unified store's `diveIntoFile` action.
- The `path` (currentNodePath) derivation: filter breadcrumb for `type === 'node'` entries
  and extract IDs. This preserves compatibility with RenderApi which expects `string[]`.
